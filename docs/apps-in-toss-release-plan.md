# Money Hunter Apps in Toss 출시 계획

## 현재 위치

현재 코드는 GCP Cloud Run에서 실행되는 심사용 웹 MVP입니다.

- `review` 프로필: 심사관과 개발자가 기능을 빠르게 확인할 수 있도록 더미 광고, 더미 결제, 테스트 패널, 게스트 유저를 허용합니다.
- `prod` 프로필: 테스트 패널, 게스트 유저, 더미 광고/결제/리워드 API를 차단합니다.
- 실제 Apps in Toss 출시 상태: 아직 아닙니다. 토스 WebView SDK 번들, 토스 로그인, 실제 광고, 실제 토스 포인트 지급, Smart Message 연동이 필요합니다.

`/api/app/config`의 `tossReleaseReady`가 `true`가 되어야 실제 출시 후보로 볼 수 있습니다. 현재는 의도적으로 `false`입니다.

## 공식 문서 기준 주요 차이

Apps in Toss는 WebView 또는 React Native SDK로 빌드한 번들을 콘솔에 업로드하는 방식입니다.
현재 정적 화면은 Spring Boot 안에 포함되어 있으므로, 최종 제출 전에는 `@apps-in-toss/web-framework` 기반 WebView 프로젝트 또는 별도 프론트 번들로 분리해야 합니다.

필수로 반영할 항목은 다음과 같습니다.

- WebView 번들: `granite.config.ts`에 `appName=money-hunter`, 브랜드명, 아이콘, WebView 설정을 넣고 `.ait` 번들을 생성합니다.
- 토스 로그인/userKey: 클라이언트는 인가 코드만 받고, 토큰 교환과 userKey 저장은 서버에서 처리합니다.
- 리워드 광고: 더미 광고 대신 Apps in Toss 광고 SDK로 사전 로드, 노출, 완료 결과를 확인한 뒤 서버에 보상 지급을 요청합니다.
- 배너 광고: 더미 배너 대신 WebView 배너 광고 SDK를 사용하고, 광고임을 명확히 유지합니다.
- 토스 포인트 지급: 골드는 게임 내 재화로 유지하고, 보상 수령 시 토스 프로모션 API 또는 게임 프로모션 SDK를 통해 지급합니다. 중복 지급 방지 키를 반드시 유지합니다.
- Smart Message: 자동사냥 종료 알림은 현재 DB 내부 알림만 생성합니다. 실제 알림은 토스 Smart Message API와 템플릿 코드가 필요합니다.
- 공유 리워드: 현재 친구 초대는 테스트 지급입니다. 출시 전에는 토스 공유 리워드 플로우와 1인 3명 제한 검증이 필요합니다.
- 인앱 결제: 동료 펫과 SP 팩은 현재 더미 결제입니다. 실제 결제 SDK 결과 검증과 서버 영수증 검증이 필요합니다.

## 출시까지 순서

1. 앱 정보 확정
   - 앱 이름: 머니헌터
   - appName: `money-hunter`
   - 부제, 상세 설명, 검색 키워드, 로고, 다크모드 로고, 썸네일, 스크린샷 등록
   - 게임 등급분류 신청 및 증빙 준비

2. WebView 프론트 전환
   - 기존 `src/main/resources/static` 화면을 Apps in Toss WebView 프로젝트로 이동
   - `@apps-in-toss/web-framework` 설치
   - `granite.config.ts` 설정
   - 세로형 화면, Safe Area, 닫기 버튼, 10초 이내 진입, 메모리 사용량 점검

3. 인증 연동
   - `appLogin`으로 인가 코드 획득
   - 서버에서 토큰 교환
   - 서버 DB의 `Player.userKey`를 토스 userKey로 저장
   - 쿠키 의존 없이 토큰 기반 인증 적용

4. 광고 연동
   - 리워드 광고 사전 로드
   - 자동사냥, 공속버프, SP 광고 보상은 광고 완료 이벤트 이후 서버 반영
   - 배너 광고는 하단 탭 위에만 노출하고 광고 UI를 임의 변경하지 않음
   - 샌드박스와 토스앱 QR 테스트에서 노출, 종료 복귀, 보상 지급 확인

5. 리워드와 포인트 지급 연동
   - 골드 100 = 토스 포인트 1 기준 유지
   - 보상 수령 전 광고 완료 확인
   - 프로모션 코드 테스트로 최소 1회 이상 지급 검증
   - 중복 호출, 재시도, 예산 소진, 지급 실패 상태 처리

6. 결제 연동
   - 동료 펫 1마리당 4,900원, 최대 2마리
   - SP 10개 999원
   - 결제 성공, 취소, 실패, 복구, 기기 변경 후 아이템 유지 확인

7. Smart Message 연동
   - 자동사냥 종료 템플릿 생성
   - 종료 예정 시점이 연장되면 기존 예약을 취소하거나 새 종료 시점 기준으로만 발송
   - 테스트 메시지 API로 실제 기기 수신 확인

8. QA와 심사
   - GCP review URL로 서버 기능 확인
   - Apps in Toss 샌드박스 앱 테스트
   - `.ait` 번들 업로드 후 토스앱 QR 테스트 최소 1회
   - 광고, 결제, 포인트 지급, 푸시, 공유 리워드, 재접속 데이터 유지 확인
   - 검토 요청

9. 운영 배포
   - 운영 Cloud Run은 `prod` 프로필로 배포
   - 더미 API가 409 또는 404로 차단되는지 확인
   - CORS 허용 Origin에 `https://money-hunter.apps.tossmini.com` 및 QR 테스트 도메인 추가
   - 출시 후 오류 로그, API 실패율, 리워드 지급 실패율 모니터링

## 출시 전 수동 QA 체크

- 앱 최초 진입이 10초 이내인지
- 직업 선택 후 데이터가 유지되는지
- 자동사냥 중 앱을 닫았다가 돌아와도 골드가 정산되는지
- 자동사냥 종료 알림이 종료 시점에 맞게 발송되는지
- 종료 전 자동사냥을 연장하면 새 종료 시점 기준으로 알림이 발송되는지
- 리워드 광고를 끝까지 본 경우에만 보상이 지급되는지
- 보상 수령 버튼에서 광고 안내가 명확한지
- 토스 포인트와 골드 표기가 혼동되지 않는지
- 동료 펫 결제 금액과 실제 결제창 금액이 일치하는지
- 결제 취소, 실패 후 아이템이 지급되지 않는지
- 친구 초대 보상이 3명까지만 지급되는지
- 운영에서 `/api/player/test/**`가 404인지
- 운영에서 더미 광고/결제/리워드 API가 차단되는지
